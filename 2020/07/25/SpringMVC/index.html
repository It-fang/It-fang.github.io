二，springMVC框架基于组件方式的执行流程

三，@RequestMapping注解：

  1. 作用：用于建立请求URL和处理请求方法之间的对应关系
  2. 位置：
    • 放在类上：请求URL的第一级访问目录，使我们的URL可以按照模块化管理
    • 放在方法上：请求URL的第二级访问目录
  3. 属性：
    • value：用于指定请求的URL。它和path属性的作用是一样的
    • method：用于指定请求的方式
    • params：用于指定限制请求参数的条件。它支持简单的表达式。要求请求参数的key和value必须和配置的一模一样。
      • params=｛”accountName”｝,表示请求参数必须有accountName
      • params=｛”money!100”｝,表示请求参数中的money不能是100
    • headers：用于指定限制请求消息头的条件。

以上四个属性只要出现2个或以上时，它们是与关系

四，请求参数的绑定

  1. 绑定机制

    • 表单提交的数据都是Key=Value格式的
    • SpringMVC的参数绑定过程是把表单提交的请求参数，作为控制器中方法的参数进行绑定的
    • 要求：提交表单的name和参数名称是相同的

  2. 支持的数据类型:

    • 基本数据类型和字符串类型
      • 提交表单的name和参数的名称是相同的
      • 区分大小写
    • 实体类型（JavaBean）
      • 提交表单的name和JavaBean中的属性名称需要一致
      • 如果一个JavaBean类中包含其他的引用类型，那么表单的name属性需要编写成：对象.属性；例如address.name
    • 集合数据类型（List，Map集合等）
      • JSP页面编写方式：list[0].属性，map[“one”].属性

  3. POST请求参数中文乱码的解决：配置一个过滤器来拦截请求设置编码

七，常用的几个注解

  1. @RequestParam
    • 作用：把请求中指定名称的参数给控制器中的形参赋值
    • 位置：写在参数前面
    • 属性：
      • value/name：请求参数中的名称

  • required：请求参数中是否必须提供此参数。默认值为true，表示必须提供且提供的请求参数必须与name的值相同，如果不提供或不相同将报错。

  1. @RequestBody

    • 作用：用于获取请求体内容。直接使用得到的是key=value&key=value…结构的数据，get请求方法不适用。

    • 位置：写在参数前面

    • 属性：

      • required：是否必须有请求体。默认值是true。当取值为true时，get请求方式会报错；如果取值为false时，get请求得到的是null。

  2. @PathVaribale

    • 作用：用于绑定url中的占位符。例如：请求url中/delete{id}，这个{id}就是url占位符。url支持占位符是spring3.0之后加入的。是springMVC支持rest风格URL的一个重要标志。

    • 位置：写在参数前面

    • 属性：

      • value/name：用于指定url中占位符名称,其值必须与请求路径的占位符的值相同

    • required：是否必须提供占位符

  3. @RequestHeader：【不常用】

    • 作用：用于获取请求消息头
    • 位置：写在参数前面
    • 属性：
      • value：提供消息头名称

  • required：是否必须有此消息头

  1. @CookieValue:【不常用】

    • 作用：用于把指定cookie名称的值传入控制器方法参数

    • 属性：

      • value：指定cookie的名称

    • required：是否必须有此cookie

  1. @ModelAttribute：

    • 作用：用于修饰方法和参数，是springMVC4.3版本以后新加入的

    • 位置：

      • 出现在方法上：表示当前方法会在控制器的方法执行之前，先执行。它可以修饰没有返回值的方法，也可以修饰有具有返回值的方法
      • 出现在参数上：获得指定的数据给参数赋值

    • 属性：

      • value：用于获取数据的key。key可以是POJO的属性名称，也可以是map结构的key
  • 应用场景：当表单提交数据不是完整的实体类数据时，保证没有提交数据的字段使用数据库对象原来的数据。例如：我们在编辑一个用户是，用户有一个创建信息字段，该字段的值是不允许被修改的。在提交表单数据是肯定没有此字段的内容，一旦更新会把该字段内容置为null，此时就可以使用此注解解决问题。

方法有返回值的情况

    @RequestMapping(path = "/hello")
    public String printHello(User user){
        System.out.println("执行了。。。。");
        return "success";
    }
    @ModelAttribute
    public User getParam(String username){
        User user = new User();
        //模拟从数据库中获得对象
        user.setUsername(username);
        user.setAge(20);
        user.setBirthday(new Date());
        return user;
}

方法没有返回值的情况：需要Map集合辅助，并在参数前面也加上@ModelAttribute注解

@RequestMapping(path = "/hello")
public String printHello(@ModelAttribute("abc") User user){
    System.out.println("执行了。。。。");
    return "success";
}
@ModelAttribute
public void getParam(String username, Map<String,User> map){
    User user = new User();
    //模拟从数据库中获得对象
    user.setUsername(username);
    user.setAge(20);
    user.setBirthday(new Date());
    map.put("abc",user);
}

  1. @SessionAttribute；

    • 作用：用于多次执行控制器方法间的参数共享

    • 位置：写在类上

    • 属性：

      • value：用于指定存入的属性名称

    • type：用于指定存入的数据类型

八，HiddentHttpMethodFilter过滤器【了解，不需要掌握】

  1. 作用：由于浏览器form表单只支持GET与POST请求，而DELETE，PUT等method并不支持，Spring3.0添加了一个过滤器，可以将浏览器请求改为指定的请求方式，发送给我们的控制器方法，使得支持GET，POST，PUT与DELETE请求
  2. 使用方法：
    1. 在web.xml中配置该过滤器
    2. 请求方式必须使用post请求
    3. 按照要求提供_method请求参数，该参数的取值就是我们需要的请求方式

九，ModelAndView对象

  • setViewName（value）：用于指定要跳转到的视图的名称，会调用视图解析器进行视图解析。

十一，springMVC的文件上传

  1. 在springMVC.xml中配置文件解析器：id必须为multipartResolver

        <property name="maxUploadSize" value="10485760"></property>
    </bean>

  2. 配置控制器中方法：参数MultipartFile对象的名称必须与上次文件组件的name值相同
