SpringMVC

一，springMVC的入门程序

  1. 在web.xml中配置前端控制器

    <servlet>
  <servlet-name>dispatcherServlet</servlet-name>
  <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
</servlet>
<servlet-mapping>
  <servlet-name>dispatcherServlet</servlet-name>
  <url-pattern>/</url-pattern>
</servlet-mapping>

  2. 向springMVC的配置文件导入springMVC的名称空间和约束

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:mvc="http://www.springframework.org/schema/mvc"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="
        http://www.springframework.org/schema/beans
        https://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/mvc
        https://www.springframework.org/schema/mvc/spring-mvc.xsd
        http://www.springframework.org/schema/context
        https://www.springframework.org/schema/context/spring-context.xsd">
</beans>

  3. 开启注解扫描

    <context:component-scan base-package="com.itfang.www"></context:component-scan>

  4. 在控制器类上使用@Controller注解

  5. 在被请求方法上使用@RequestMapping注解

  6. 在前端控制器中加载springMVC的配置文件并说明在第一次请求时创建前端控制器对象

    <servlet>
  <servlet-name>dispatcherServlet</servlet-name>
  <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
  <init-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:springMVC.xml</param-value>
  </init-param>
  <load-on-startup>1</load-on-startup>
</servlet>

  7. 在被请求方法的返回值处写上跳转的jsp页面名称

  8. 在springMVC的配置文件中配置视图解析器对象

     <bean id="internalResourceViewResolver" class="org.springframework.web.servlet.view.InternalResourceViewResolver">
<!--        说明要解析的文件在哪个目录下-->
        <property name="prefix" value="/WEB-INF/pages/"></property>
<!--        说明要解析的文件的后缀-->
        <property name="suffix" value=".jsp"></property>
    </bean>

  9. 在springMVC的配置文件中开启springMVC的注解配置支持

    <mvc:annotation-driven></mvc:annotation-driven>

二，springMVC框架基于组件方式的执行流程

三，@RequestMapping注解：

  1. 作用：用于建立请求URL和处理请求方法之间的对应关系
  2. 位置：
    • 放在类上：请求URL的第一级访问目录，使我们的URL可以按照模块化管理
    • 放在方法上：请求URL的第二级访问目录
  3. 属性：
    • value：用于指定请求的URL。它和path属性的作用是一样的
    • method：用于指定请求的方式
    • params：用于指定限制请求参数的条件。它支持简单的表达式。要求请求参数的key和value必须和配置的一模一样。
      • params=｛”accountName”｝,表示请求参数必须有accountName
      • params=｛”money!100”｝,表示请求参数中的money不能是100
    • headers：用于指定限制请求消息头的条件。

以上四个属性只要出现2个或以上时，它们是与关系

四，请求参数的绑定

  1. 绑定机制

    • 表单提交的数据都是Key=Value格式的
    • SpringMVC的参数绑定过程是把表单提交的请求参数，作为控制器中方法的参数进行绑定的
    • 要求：提交表单的name和参数名称是相同的

  2. 支持的数据类型:

    • 基本数据类型和字符串类型
      • 提交表单的name和参数的名称是相同的
      • 区分大小写
    • 实体类型（JavaBean）
      • 提交表单的name和JavaBean中的属性名称需要一致
      • 如果一个JavaBean类中包含其他的引用类型，那么表单的name属性需要编写成：对象.属性；例如address.name
    • 集合数据类型（List，Map集合等）
      • JSP页面编写方式：list[0].属性，map[“one”].属性

  3. POST请求参数中文乱码的解决：配置一个过滤器来拦截请求设置编码

    <filter>
  <filter-name>characterEncodingFilter</filter-name>
  <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
  <init-param>
    <param-name>encoding</param-name>
    <param-value>UTF-8</param-value>
  </init-param>
</filter>
<filter-mapping>
  <filter-name>characterEncodingFilter</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>

五，自定义类型转换器

  1. 自定义类型转换器的类实现实现Converter接口

    public class StringToDateConverter implements Converter<String,Date> {

    @Override
    public Date convert(String source) {
        if (source == null){
            throw new RuntimeException("请输入数据");
        }
       DateFormat df = new SimpleDateFormat("yyyy-MM-dd");
        try {
            return df.parse(source);
        } catch (Exception e) {
            throw new RuntimeException("格式转换出错");
        }

    }
}

  2. 在springMVC中配置类型转换器

    <!--    配置自定义类型转换器-->
    <bean id="conversionService" class="org.springframework.context.support.ConversionServiceFactoryBean">
        <property name="converters">
            <set>
                <bean class="com.itfang.www.utils.StringToDateConverter"></bean>
            </set>
        </property>
    </bean>

  3. 在开启springMVC框架支持的标签中配置conversion-service属性

    <mvc:annotation-driven conversion-service="conversionService"></mvc:annotation-driven>

七，常用的几个注解

  1. @RequestParam

    • 作用：把请求中指定名称的参数给控制器中的形参赋值

    • 位置：写在参数前面

    • 属性：

      • value/name：请求参数中的名称

      • required：请求参数中是否必须提供此参数。默认值为true，表示必须提供且提供的请求参数必须与name的值相同，如果不提供或不相同将报错。

        public String printHello(@RequestParam(name = "name") String username){

}

  2. @RequestBody

    • 作用：用于获取请求体内容。直接使用得到的是key=value&key=value…结构的数据，get请求方法不适用。

    • 位置：写在参数前面

    • 属性：

      • required：是否必须有请求体。默认值是true。当取值为true时，get请求方式会报错；如果取值为false时，get请求得到的是null。

        public String printHello(@RequestBody String body){

}

  3. @PathVaribale

    • 作用：用于绑定url中的占位符。例如：请求url中/delete{id}，这个{id}就是url占位符。url支持占位符是spring3.0之后加入的。是springMVC支持rest风格URL的一个重要标志。

    • 位置：写在参数前面

    • 属性：

      • value/name：用于指定url中占位符名称,其值必须与请求路径的占位符的值相同

      • required：是否必须提供占位符

        @RequestMapping(path = "/hello/{id}")
public String printHello(@PathVariable(name = "id")String id){

}

  4. @RequestHeader：【不常用】

    • 作用：用于获取请求消息头

    • 位置：写在参数前面

    • 属性：

      • value：提供消息头名称

      • required：是否必须有此消息头

        @RequestMapping(path = "/hello")
public String printHello(@RequestHeader("Accept")String header){

}

  5. @CookieValue:【不常用】

    • 作用：用于把指定cookie名称的值传入控制器方法参数

    • 属性：

      • value：指定cookie的名称

      • required：是否必须有此cookie 

        @RequestMapping(path = "/hello")
public String printHello(@CookieValue(value = "JSESSIONID")String cookie){

}

  6. @ModelAttribute：

    • 作用：用于修饰方法和参数，是springMVC4.3版本以后新加入的
    • 位置：
      • 出现在方法上：表示当前方法会在控制器的方法执行之前，先执行。它可以修饰没有返回值的方法，也可以修饰有具有返回值的方法
      • 出现在参数上：获得指定的数据给参数赋值
    • 属性：
      • value：用于获取数据的key。key可以是POJO的属性名称，也可以是map结构的key
    • 应用场景：当表单提交数据不是完整的实体类数据时，保证没有提交数据的字段使用数据库对象原来的数据。例如：我们在编辑一个用户是，用户有一个创建信息字段，该字段的值是不允许被修改的。在提交表单数据是肯定没有此字段的内容，一旦更新会把该字段内容置为null，此时就可以使用此注解解决问题。

方法有返回值的情况

@RequestMapping(path = "/hello")
public String printHello(User user){
    System.out.println("执行了。。。。");
    return "success";
}
@ModelAttribute
public User getParam(String username){
    User user = new User();
    //模拟从数据库中获得对象
    user.setUsername(username);
    user.setAge(20);
    user.setBirthday(new Date());
    return user;
}

方法没有返回值的情况：需要Map集合辅助，并在参数前面也加上@ModelAttribute注解

@RequestMapping(path = "/hello")
public String printHello(@ModelAttribute("abc") User user){
    System.out.println("执行了。。。。");
    return "success";
}
@ModelAttribute
public void getParam(String username, Map<String,User> map){
    User user = new User();
    //模拟从数据库中获得对象
    user.setUsername(username);
    user.setAge(20);
    user.setBirthday(new Date());
    map.put("abc",user);
}

  1. @SessionAttribute；

    • 作用：用于多次执行控制器方法间的参数共享

    • 位置：写在类上

    • 属性：

      • value：用于指定存入的属性名称

      • type：用于指定存入的数据类型

        @Controller
@SessionAttributes(value = {"msg"})//将model存在request域中的值存入session域中
public class HelloWorldController {
    @RequestMapping(path = "/testSetRequestAttritube")
    public String testSetRequestAttritube(Model model){
//        将消息以键值对的形式存入request域中
        model.addAttribute("msg", "你好帅");
        return "success";
    }
    @RequestMapping(path = "/testGetSessionAttritube")
    public String testGetSessionAttritube(ModelMap modelMap){
        modelMap.get("msg");
        return "success";
    }
    @RequestMapping(path = "/testDelSessionAttritube")
    public String testDelSessionAttritube(SessionStatus status){
        status.setComplete();//清除session域中的值
        return "success";
    }
}

八，HiddentHttpMethodFilter过滤器【了解，不需要掌握】

  1. 作用：由于浏览器form表单只支持GET与POST请求，而DELETE，PUT等method并不支持，Spring3.0添加了一个过滤器，可以将浏览器请求改为指定的请求方式，发送给我们的控制器方法，使得支持GET，POST，PUT与DELETE请求
  2. 使用方法：
    1. 在web.xml中配置该过滤器
    2. 请求方式必须使用post请求
    3. 按照要求提供_method请求参数，该参数的取值就是我们需要的请求方式

九，ModelAndView对象

  1. 作用：作为控制器的返回值携带数据跳转到指定视图

  2. 方法：

    • addObject（key，value）：以键值对的形式将数据存入到request域中

    • setViewName（value）：用于指定要跳转到的视图的名称，会调用视图解析器进行视图解析。

      @RequestMapping("/hello")
    public ModelAndView testModelAndView(){
        ModelAndView mv = new ModelAndView();
        //模拟查询User对象
        User user = new User();
        user.setUsername("小方");
        user.setPassword("123456");
        user.setAge(20);
//        将user对象存入到mv中,最终mv对象会将user对象存入到request域中
        mv.addObject("user",user);
        mv.setViewName("success");
        return mv;
    }

十一，springMVC的文件上传

  1. 在springMVC.xml中配置文件解析器：id必须为multipartResolver

    <!--    对象的id必须为multipartResolver-->
    <bean id="multipartResolver" class="org.springframework.web.multipart.commons.CommonsMultipartResolver">
<!--        配置上传文件的最大格式-->
        <property name="maxUploadSize" value="10485760"></property>
    </bean>

  2. 配置控制器中方法：参数MultipartFile对象的名称必须与上次文件组件的name值相同

    @RequestMapping("/fileUpload")
    public String fileUpload(HttpServletRequest request, MultipartFile upload) throws Exception {
        //上传文件的位置
        String path = request.getSession().getServletContext().getRealPath("/uploads/");
        System.out.println(path);
//        判断文件的目录是否存在
        File file = new File(path);
        if (!file.exists()){
            file.mkdirs();
        }
//        获得上传文件的名称
        String fileName = upload.getOriginalFilename();
//        给文件的名称加上唯一ID
        String uuid = UUID.randomUUID().toString().replace("-","");
        fileName = uuid+"_"+fileName;
        upload.transferTo(new File(path,fileName));
        System.out.println("文件上传了");
        return "success";
    }

十二，springMVC的异常处理

  1. 编写自定义异常类（做提示信息）

    public class SysException extends Exception {
    private String msg;

    public SysException(String msg) {
        this.msg = msg;
    }

    public String getMsg() {
        return msg;
    }

    public void setMsg(String msg) {
        this.msg = msg;
    }
}

  2. 编写异常处理器：要实现HandlerExceptionResolver 接口

    public class SysExceptionResolver implements HandlerExceptionResolver {
    @Override
    public ModelAndView resolveException(HttpServletRequest httpServletRequest, HttpServletResponse httpServletResponse, Object o, Exception e) {
        SysException sysException = null;
        if ( e instanceof SysException){
            sysException = (SysException) e;
        }else {
            sysException = new SysException("系统出错了");
        }
        ModelAndView mv = new ModelAndView();
        mv.addObject("msg",sysException.getMsg());
        mv.setViewName("error");
        return mv;

    }
}

  3. 配置异常处理器

    <!--    配置异常处理器-->
    <bean id="sysExceptionResolver" class="com.itfang.www.utils.SysExceptionResolver"></bean>

十三，springMVC的拦截器

  1. 编写拦截器类：实现HandlerInterceptor接口

    public class MyInterceptor implements HandlerInterceptor {
    /**
     *预处理：在控制器方法执行前执行
     * return true：表示放行,执行下一个拦截器,如果没有,则执行控制器方法
     * return false：不放行
     * @param request
     * @param response
     * @param handler
     * @return
     * @throws Exception
     */
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("执行了拦截器.....前");
        return true;
    }

    /**
     * 后处理：在控制器方法执行后执行
     * @param request
     * @param response
     * @param handler
     * @param modelAndView
     * @throws Exception
     */
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("执行了拦截器.....后");
    }

    /**
     * 在jsp页面执行完之后执行
     * @param request
     * @param response
     * @param handler
     * @param ex
     * @throws Exception
     */
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("执行了拦截器.....jsp后");
    }
}

  2. 配置拦截器

        <mvc:interceptors>
<!--        配置拦截器-->
        <mvc:interceptor>
<!--            配置拦截的方法,没有配置的方法不拦截
                如果path="/**"表示所有控制器的方法全拦截
                如果path="/user/*"表示user控制器下的方法全拦截
-->
            <mvc:mapping path="/user/*"/>
<!--            配置不拦截的方法,没有配置的方法全拦截-->
<!--            <mvc:exclude-mapping path=""/>-->
<!--            配置拦截器对象-->
            <bean class="com.itfang.www.utils.MyInterceptor"></bean>
        </mvc:interceptor>
    </mvc:interceptors>
