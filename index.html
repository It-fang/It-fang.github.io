二，springMVC框架基于组件方式的执行流程

三，@RequestMapping注解：

  1. 作用：用于建立请求URL和处理请求方法之间的对应关系
  2. 位置：
    • 放在类上：请求URL的第一级访问目录，使我们的URL可以按照模块化管理
    • 放在方法上：请求URL的第二级访问目录
  3. 属性：
    • value：用于指定请求的URL。它和path属性的作用是一样的
    • method：用于指定请求的方式
    • params：用于指定限制请求参数的条件。它支持简单的表达式。要求请求参数的key和value必须和配置的一模一样。
      • params=｛”accountName”｝,表示请求参数必须有accountName
      • params=｛”money!100”｝,表示请求参数中的money不能是100
    • headers：用于指定限制请求消息头的条件。

以上四个属性只要出现2个或以上时，它们是与关系

四，请求参数的绑定

  1. 绑定机制

    • 表单提交的数据都是Key=Value格式的
    • SpringMVC的参数绑定过程是把表单提交的请求参数，作为控制器中方法的参数进行绑定的
    • 要求：提交表单的name和参数名称是相同的

  2. 支持的数据类型:

    • 基本数据类型和字符串类型
      • 提交表单的name和参数的名称是相同的
      • 区分大小写
    • 实体类型（JavaBean）
      • 提交表单的name和JavaBean中的属性名称需要一致
      • 如果一个JavaBean类中包含其他的引用类型，那么表单的name属性需要编写成：对象.属性；例如address.name
    • 集合数据类型（List，Map集合等）
      • JSP页面编写方式：list[0].属性，map[“one”].属性

  3. POST请求参数中文乱码的解决：配置一个过滤器来拦截请求设置编码

    public class StringToDateConverter implements Converter<String,Date> {

    @Override
    public Date convert(String source) {
        if (source == null){
            throw new RuntimeException("请输入数据");
        }
       DateFormat df = new SimpleDateFormat("yyyy-MM-dd");
        try {
            return df.parse(source);
        } catch (Exception e) {

        }

七，常用的几个注解

  1. @RequestParam

    • 作用：把请求中指定名称的参数给控制器中的形参赋值

    • 位置：写在参数前面

    • 属性：

      • value/name：请求参数中的名称

      • required：请求参数中是否必须提供此参数。默认值为true，表示必须提供且提供的请求参数必须与name的值相同，如果不提供或不相同将报错。

        public String printHello(@RequestParam(name = "name") String username){

}

  2. @RequestBody

    • 作用：用于获取请求体内容。直接使用得到的是key=value&key=value…结构的数据，get请求方法不适用。

    • 位置：写在参数前面

    • 属性：

      • required：是否必须有请求体。默认值是true。当取值为true时，get请求方式会报错；如果取值为false时，get请求得到的是null。

        public String printHello(@RequestBody String body){

}

  3. @PathVaribale

    • 作用：用于绑定url中的占位符。例如：请求url中/delete{id}，这个{id}就是url占位符。url支持占位符是spring3.0之后加入的。是springMVC支持rest风格URL的一个重要标志。

    • 位置：写在参数前面

    • 属性：

      • value/name：用于指定url中占位符名称,其值必须与请求路径的占位符的值相同

      • required：是否必须提供占位符

        @RequestMapping(path = "/hello/{id}")
public String printHello(@PathVariable(name = "id")String id){

}

  4. @RequestHeader：【不常用】

    • 作用：用于获取请求消息头

    • 位置：写在参数前面

    • 属性：

      • value：提供消息头名称

      • required：是否必须有此消息头

        @RequestMapping(path = "/hello")
public String printHello(@RequestHeader("Accept")String header){

}

  5. @CookieValue:【不常用】

    • 作用：用于把指定cookie名称的值传入控制器方法参数

    • 属性：

      • value：指定cookie的名称

      • required：是否必须有此cookie 

        @RequestMapping(path = "/hello")
public String printHello(@CookieValue(value = "JSESSIONID")String cookie){

}

  6. @ModelAttribute：

    • 作用：用于修饰方法和参数，是springMVC4.3版本以后新加入的
    • 位置：
      • 出现在方法上：表示当前方法会在控制器的方法执行之前，先执行。它可以修饰没有返回值的方法，也可以修饰有具有返回值的方法
      • 出现在参数上：获得指定的数据给参数赋值
    • 属性：
      • value：用于获取数据的key。key可以是POJO的属性名称，也可以是map结构的key
    • 应用场景：当表单提交数据不是完整的实体类数据时，保证没有提交数据的字段使用数据库对象原来的数据。例如：我们在编辑一个用户是，用户有一个创建信息字段，该字段的值是不允许被修改的。在提交表单数据是肯定没有此字段的内容，一旦更新会把该字段内容置为null，此时就可以使用此注解解决问题。

方法有返回值的情况

@RequestMapping(path = "/hello")
public String printHello(User user){
    System.out.println("执行了。。。。");
    return "success";
}

spring

一，ApplicationContext的三个常用实现类：

  1. ClassPathXmlApplicationContext：它可以加载类路径下的配置文件，要求配置文件必须在类路径下，否则加载不了。
  2. FileSystemXmlApplicationContext：它库加载磁盘任意路径下的配置文件（必须要有访问权限）
  3. AnnotationConfigApplicationContext：它是用于读取注解创建容器的。

二，Ioc核心容器的两个接口引发的问题：

  1. ApplicationContext：它在构建核心容器时，创建对象采取的策略是采用立即加载的方式；也就是说，只要一读取完配置文件马上就创建配置文件中配置的对象。【单例对象下适用】【常用】
  2. BeanFactory：它在构建核心容器时，创建对象采取的策略是采用延迟加载的方式；也就是 说，什么时候根据id获取对象了，什么时候才真正的创建对象。【多例对象下适用】

三，spring创建Bean对象的三种方式：

  1. 使用默认构造函数创建：在spring的配置文件中使用bean标签，配以id和class属性之后，且没有其他属性和标签时，采用的就是默认构造函数创建bean对象，此时如果类中没有默认构造函数，则对象无法创建 

    <bean id="accountService" class="com.itfang.www.service.AccountServiceImpl"></bean>

  2. 使用普通工厂中的方法创建对象（使用某个类中的方法创建对象，并存入spring容器中） 

    <bean id="instanceFactory" class="com.itfang.www.factory.InstanceFactory"></bean>
<bean id="accountService" factory-bean="instanceFactory" factory-method="getAccountService"></bean>

  3. 使用普通工厂中的静态方法创建对象（使用某个类中的静态方法创建对象，并存入spring容器中）

  4. <bean id="accountService" class="com.itfang.www.factory.StaticFactory" factory-method="getAccountService"></bean>

四，spring中bean的作用范围：

  • bean标签的scope属性：
    1. singleton：单例的（默认值）【常用】
    2. prototype：多例的【常用】
    3. request：作用于web应用的请求范围
    4. session：作用于web应用的会话范围
    5. ** global-session：**作用于集群环境的会话范围（全局会话范围），当不是集群环境时，它就相当于session

五，spring中bean对象的生命周期:

  • 单例对象：
    1. 创建：当Ioc容器创建时，对象创建
    2. 存在：当容器存在时，对象一直存在
    3. 销毁：当容器销毁时，对象销毁
  • 多例对象：
    1. 创建：当使用对象时spring框架才为我们创建
    2. 存在：对象只要在使用过程中就一直存在
    3. 销毁：当对象长时间不用，且没有别的对象引用时，由Java的垃圾回收机制回收

六，spring的依赖注入：

  1. 构造函数注入：
    • 使用的标签：constructor-arg
    • 标签出现的位置：bean标签内部
    • 标签的属性
      1. type：用于指定要注入的数据的数据类型，该数据类型也是构造函数中某个 或某些参数的类型
      2. index：用指定要注入的数据给构造函数中指定索引位置的参数赋值。索引的位置是从0开始
      3. name：用于指定给构造函数汇总指定名称的参数赋值。【常用】
      4. value：用于提供基本类型和String类型的数据。
      5. ref：用于指定其他bean类型数据。它指的是在spring的Ioc核心容器中出现过的bean对象
    • 优势：在获取bean对象时，必须按照构造函数的参数注入数据，否则对象无法创建成功
    • 弊端：改变了bean对象的实例化方式，使我们在创建对象时，如果用不到这些数据，也必须提供。
  2. set方法注入：【常用】
    • 使用的标签：property
    • 标签出现的位置：bean标签内部
    • 标签的属性：
      1. name：用于指定注入时的类属性名
      2. value：用于提供基本类型和String类型的数据。
      3. ref:用于指定其他bean类型数据。它指的是在spring的Ioc核心容器中出现过的bean对象
    • 优势：创建对象时没有明确的限制，可以直接使用构造函数
    • 弊端：如果有某个成员变量必须有值，则获取对象是有可能set方法没有执行

七，spring给复杂类型/集合类型依赖注入时，只要结构相同，标签可以互换

  • 比如给list，arraylist，set注入时，可以用list，array，set标签；
  • 比如给map，props注入时，可以用map,props标签。

  1. 告知spring扫描的包并导入约束

  2. 用于创建对象：

    • @Component：把当前类对象存入spring容器中
      • value属性：用于指定bean的id，当我们不写时，它的默认值就是当前类名，且首字母改为小写

    以下三个注解的作用和属性跟@Component标签几乎一样，是用来使三层对象更加清晰

    • @Controller：一般用在表现层
    • @Service：一般用在业务层
    • @Repository：一般用在持久层

  3. 用于注入数据：

    • @Autowired：自动按照类型注入。
      • 只要容器中有唯一的一个bean对象类型和要注入的变量类型匹配，就可以注入成功。
      • 如果IOC容器中没有任何bean的类型和要注入的变量类型匹配，则报错；
      • 如果IOC容器中有多个类型匹配时，则根据bean的id和要注入的变量名称匹配。
    • @Qualifier：在按照类中注入的基础之上再按照名称注入。它在给类成员注入时不能单独使用，需要和@Autowired标签配合使用，但是在给方法参数注入时可以单独使用。
      • value属性：用于指定注入bean的id
    • @Resource：直接按照bean的id注入，它可以单独使用
      • name属性：用于指定注入bean的id

    以上三个注入都只能注入其他bean类型的数据，而基本类型和String类型无法使用上述注解实现。另外，集合类型的注入只能通过XML来实现。

    • @Value：用于注入基本类型和String类型的数据
      • value属性：用于指定数据的值。它可以使用spring中的SpEL（也就是spring的EL表达式），写法：${表达式}

  4. 用于改变作用范围：

    • @Scope：用于指定bean的作用范围
      • value属性：指定范围的取值。常用取值：singleton【默认】，propotype

  5. 用于改变生命周期：

    • @PostConstruct：用于指定初始化方法
    • @PostDestroy：用于指定销毁方法

九，spring的配置类：

  • @Configuration：指定当前类是一个配置类

    • 细节：当配置类作为AnnotationConfigApplicationContext对象创建的参数时，该注解可以不写

  • @ComponentScan：用于通过注解指定spring在创建容器时要扫描的包

    • value属性：它和basePackages的作用是一样的，都是用于指定创建容器时要扫描的包。

  • @Bean：用于把当前方法的返回值作为bean对象存入spring的IOC容器中

    • name属性：用于指定bean的id。当不写时，默认值是当前方法的名称。
    • 细节：当我们使用注解配置方法时，如果方法有参数，spring框架会去容器中查找有没有可用的bean对象；查找的方式和Autowired注解的作用是一样的。

  • 使用配置类后，ApplicationContext不再使用ClassPathXmlApplicationContext来读取配置文件，而是通过AnnotationConfigApplicationContext来读取配置类。

  • @Import：用于导入其他的配置类

  • @PropertySource：用于指定properties文件的位置

    • value属性：指定文件的名称和路径；关键字：classpath，表示类路径下

  • 基于接口的动态代理

    1. 创建：使用Proxy类中的newProxyInstance方法

    2. 创建的要求：被代理类最少实现一个接口，如果没有则不能使用

    3. newProxyInstance方法的参数：

      1. ClassLoader：它是用于加载代理对象字节码，和被代理对象使用相同的类加载器

      2. Class[]：它是用于让代理对象和被代理对象有相同方法。

      3. InvocationHandler：它是让我们写如何代理。我们一般都是写一个该接口的实现类，通常情况下都是匿名内部类【匿名内部类访问外部成员变量时，要求外部成员变量是final修饰的】，但不是必须的。此接口的实现类谁用谁写。

        public static void main(String[] args){
    IProducer producer = new Producer();
    IProducer proxyProducer = (IProducer)Proxy.newProxyInstance(producer.getClass().getClassLoader(),
            producer.getClass().getInterfaces(),
            new InvocationHandler() {
                @Override
                public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                    Float money = (Float)args[0];
                    Object rtValue = null;
                    if ("sale".equals(method.getName())){
                        rtValue = method.invoke(producer, money * 0.8f);
                    }
                    return rtValue;
                }
            });
    proxyProducer.sale(10000f);
}

十一，AOP的相关术语

  • Joinpoint（连接点）：所谓连接点是指那些被拦截到的点。在spring中，这些点指的是方法，因为spring只支持方法类型的连接点。
  • Pointcut（切入点）：所谓切入点是指我们要对哪些Joinpoint进行拦截的定义。
  • Advice（通知/增强）：所谓通知是指拦截到Joinpoint之后所要做的事情就是通知。通知的类型：前置通知，后置通知，异常通知，最终通知，环绕通知
  • Introduction（引介）：引介是一种特殊的通知在不修改类代码的前提下，Introduction可以在运行期为类动态地添加一些方法或Field.。
  • Target（目标对象）：代理的目标对象
  • Weaving（织入）：是指把增加应用到目标对象来创建新的代理对象的过程。spring采用动态代理织入，而AspectJ采用编译期织入和类加载期织入。
  • Proxy（代理）：一个类被AOP织入增强后，就产生一个结果代理类。
  • Aspect（切面）：是切入点和通知（引介）的结合。

十二，spring中基于XML的AOP配置步骤：

  1. 把通知Bean也交给spring的IOC容器来管理

  2. 使用aop:config标签表明开始AOP的配置

  3. 配置切点表达式

    • 关键字：execution（表达式）
    • 标准的表达式：访问修饰符 返回值 全限定类名.方法名（参数列表）
      • 访问修饰符可以省略不写
      • 返回值可以使用通配符 * 来表示任意返回值
      • 包名可以使用通配符 * 来表示任意包，但是有几级包，就需要写几个 * .也可以使用 *.. 来表示当前包及其子包
      • 类名和方法名可以使用通配符 * 来实现通配
      • 参数列表：可以直接写数据类型
        • 基本类型直接写名称（int）
        • 引用类型写包名.类名（java.lang.String）
        • 可以使用通配符 * 表示任意类型，但切入的方法必须有参数，且有几个参数就要写几个*
        • 可以使用..表示有无参数均可，有参数可以表示任意类型
    • 全通配表达式： * *. . * . * (..)
    • 实际开发中切入点表达式的通常写法：切到业务层实现类下的所有方法
        • com.itfang.www.service.impl. * . * (..)

  4. 使用aop:aspect标签表明配置切面

    • id属性：是给切面提供一个唯一标识
    • ref属性：是指定通知类bean的id

  5. 在aop:aspect标签的内部使用对应标签来配置通知类型

    • aop:before：表示配置前置通知

      • method属性：用于指定通知类中哪个方法是前置通知

      • pointcut属性：用于指定切入点表达式，该表达式的含义指的是将哪个连接点指定为切入点来获得增强 

        <bean id="logger" class="com.itfang.www.utils.Logger"></bean>

十三，spring中基于注解的AOP配置：注解的AOP会先调用最终通知再调用后置通知（异常通知），解决方法：配置成环绕通知

  1. 配置切面类@Aspect

  2. 配置切入点表达式@Pointcut

  3. 配置前置通知@Before（org.aspectj.lang.annotation.Before）

  4. 配置后置通知@AfterReturning

  5. 配置异常通知@AfterThrowing

  6. 配置最终通知@After

  7. 在XML中开启AOP注解配置

    <aop:aspectj-autoproxy></aop:aspectj-autoproxy>
@Aspect
public class Logger {
    /**
     * 切入点表达式
     */
    @Pointcut("execution(* com.itfang.www.service.Impl.AccountServiceImpl.*(..))")
    private void pt1(){}

    /**
     * 前置通知
     */
    @Before("pt1()")
    public void printLog(){
        System.out.println("前置通知");
    }
    /**
     * 后置通知
     */
    @AfterReturning("pt1()")
    public void afterReturn(){
        System.out.println("后置通知");
    }
    /**
     * 异常通知
     */
    @AfterThrowing("pt1()")
    public void afterThrowing(){
        System.out.println("异常通知");
    }
    /**
     * 最终通知
     */
    @After("pt1()")
    public void after(){
        System.out.println("最终通知");
    }
}

十四，spring中环绕通知的配置

  1. 注明方法为环绕通知的两种方式:

    • 配置XML

      <bean id="logger" class="com.itfang.www.utils.Logger"></bean>
<aop:config >
    <aop:aspect id="logAdvice" ref="logger">
        <aop:around method="aroundPrintLog" pointcut="execution(public void com.itfang.www.service.Impl.AccountServiceImpl.save())"></aop:around>
    </aop:aspect>
</aop:config>

    • 在切面类中用注解@Around注明为环绕通知

  2. 配置方法

    public Object aroundPringLog(ProceedingJoinPoint proceedingJoinPoint){
    Object rtValue = null;
    try{
        Object args[] = proceedingJoinPoint.getArgs();
        System.out.println("前置");
        rtValue = proceedingJoinPoint.proceed(args);
        System.out.println("后置");
        return rtValue;
    }catch (Throwable t){//此处异常必须使用Throwable，不能用Execption
        System.out.println("异常");
        throw new RuntimeException(t);
    }finally {
        System.out.println("最终");
    }
}

十五，spring中基于XML的声明式事务控制配置步骤

  1. 配置事务管理器

  2. 配置事务的通知：需要导入事务的约束，tx和aop名称空间和约束

    • 使用tx:advice标签配置事务通知
      • id：给事务通知起一个唯一标识
      • transaction-manager：给事务通知提供一个事务管理器引用

  3. 配置AOP中的通用切入点表达式

  4. 建立事务通知和切入点表达式的对应关系

  5. 配置事务的属性

    • 位置：在事务的通知tx:advice标签内部

    • 属性：

      • name：获得事务管理的方法名

      • isolation：用于指定事务的隔离级别。默认值是DEFAULT，表示使用数据库的默认隔离级别

      • propagation：用于指定事务的传播行为。默认值是REQUIRED，表示一定会有事务，增删改的选择。查询方法可以选择SUPPORTS。

      • read-only：用于指定事务是否只读。只有查询方法才能设置为true。默认值是false，表示读写。

      • timeout：用于指定事务的超时时间，默认值是-1，表示永不超时。如果指定了数值，以秒为单位。

      • rollback-for；用于指定一个异常，当产生该异常时，事务回滚，产生其他异常时，事务不回滚。没有默认值，表示任何异常都回滚

      • no-rollback-for：用于指定一个异常，当产生该异常时，事务不回滚，产生其他异常时事务回滚。没有默认值，表示任何异常都回滚。

        <bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="dataSource"></property>
</bean>

<tx:advice id="txAdvice" transaction-manager="transactionManager">
    <tx:attributes>
        <tx:method name="*" propagation="REQUIRED" read-only="false"/>
        <tx:method name="find*" propagation="SUPPORTS" read-only="true"></tx:method>
    </tx:attributes>
</tx:advice>

    <aop:pointcut id="pt" expression="execution(* com.itfang.www.service.impl.AccountServiceImpl.*(..)))"/>
    <aop:advisor advice-ref="txAdvice" pointcut-ref="pt"></aop:advisor>
</aop:config>
